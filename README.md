# ローン管理アプリ

## 実装済み機能

- 複数ローンの登録・編集・削除
- 元利均等返済
- 元金均等返済
- 一括返済
- ボーナス払い（元利均等返済・元金均等返済との併用）
- 固定金利（百分率入力、小数点以下桁数はJSON設定）
- 月割り計算 / 日割り計算の選択
- 金額の端数処理は1円未満を四捨五入
  - 50銭未満は切り捨て
  - 50銭以上1円未満は1円へ切り上げ
- 返済期間または毎月の金額による返済条件入力
- 月末超過時の月末調整
- 土日の前営業日調整（祝日は考慮しない）
- 月ごとの返済額（元金）・利息・お支払い額・残高表示
- 残りの返済スケジュール表示
- 日付シミュレーション（次回返済・推定残高・残回数・スケジュール状態を選択日基準で表示）
- 入金失敗の登録・解消
- SQLiteによるローカル保存
- JSON設定
- 起動日時ごとのログファイル

- Visual Studio 2022

- C# 7.3
- x64

## ビルド方法

1. `LoanManagerApp.sln` をVisual Studio 2022で開きます。
2. NuGetパッケージの復元を実行します。
3. 構成を `Release / x64` にします。
4. ソリューションをビルドします。

または、Visual Studio 2022 / .NET SDKが利用できる環境で `BuildRelease.bat` を実行すると、`dist\LoanManagerApp_x64.zip` まで生成します。

NuGetで以下が復元されます。

- `System.Data.SQLite.Core` 1.0.119
- `Microsoft.NETFramework.ReferenceAssemblies.net48` 1.0.3

顧客PCにSQLiteを別途インストールする必要はありません。ただし、Release出力フォルダ内のSQLite関連DLL・ネイティブDLLを含めて、フォルダ一式を配布してください。

## データ保存場所

設定・ローンデータ・ログは、初回起動時に `LoanManagerApp.exe` と同じ階層へ自動作成します。

```text
配布フォルダ\
├─ LoanManagerApp.exe
├─ LoanManagerApp.exe.config
├─ Config\
│  └─ settings.json
├─ Data\
│  └─ loan.db
├─ Logs\
│  └─ yyyyMMdd_HHmmss.log
└─ その他の依存DLL
```

同じ秒に複数回起動した場合、ログ名は次のようになります。

```text
20260710_123456.log
20260710_123456_01.log
```

`Logs` フォルダは存在しない場合に起動時に自動作成します。ログを書き込むため、配布フォルダには利用者の書き込み権限が必要です。`Program Files` 直下など書き込みが制限された場所への配置は避けてください。

## settings.json 初期値

```json
{
  "interestRateDecimalPlaces": 2,
  "minimumLoanAmount": 1000,
  "maximumLoanAmount": 1000000000,
  "minimumRepaymentMonths": 1,
  "maximumRepaymentMonths": 600,
  "minimumInterestRate": 0.00,
  "maximumInterestRate": 99.99
}
```


端数処理は設定ファイルでは変更できません。すべて1円未満の四捨五入で計算します。

設定ファイルが破損している場合は、破損ファイルを `.broken_yyyyMMdd_HHmmss` として退避し、初期値で再生成します。

## 現時点の計算仕様

### 端数処理

計算途中で1円未満が生じた金額は、50銭未満を切り捨て、50銭以上1円未満を1円へ切り上げます。端数処理方式は固定で、画面やJSONから変更できません。

### 月割り計算

```text
当月利息 = 返済前残高 × 年利 ÷ 12
```

### 日割り計算

Actual/Actual方式です。年をまたぐ場合は期間を年単位に分割し、通常年は365日、うるう年は366日として計算します。

### 一括返済

- 返済スケジュールは1行です。
- 日割り計算では借入日から初回返済日までを計算します。
- 月割り計算では入力された返済期間（月数）分の利息を計算します。

### ボーナス払い

ボーナス払いは返済方式ではなく、元利均等返済または元金均等返済に追加する任意設定です。

- 「年1回」または「年2回」を選択した場合だけ設定欄を表示します。
- ボーナス払い対象元金は、指定月ごとに通常返済へ上乗せする1回あたりの元金額です。
- 通常返済は借入残高全体を基準に計算し、ボーナス月だけ追加元金を加算します。
- 入力した金額は回数で分割せず、指定した各月に同額を加算します。
- 利息は返済前残高全体に対して各月計算し、ボーナス分だけを別枠で累積しません。
- 一括返済ではボーナス払いを使用できません。
- 最終回に残額を調整して残高を0円にします。

返済スケジュールの表示は次の意味です。

- **返済額（元金）**: その回に返済する元金。残高から減る金額
- **利息**: 返済前残高と金利から計算した金額
- **お支払い額**: 返済額（元金）と利息の合計。ボーナス月はボーナス分も含む金額

この方式を金融機関固有の計算方法へ変更する場合は、`Services/RepaymentCalculator.cs` のボーナス計算部分を変更します。


### 返済条件の設定方法

- **返済期間で設定**: 年・月を入力し、選択した返済方式に基づいて返済額を計算します。
- **毎月の金額で設定**: 金額を入力し、最大600か月の範囲で完済までの期間を自動算出します。
  - 元利均等返済では、入力額を元金と利息を含む毎月のお支払い額として使用します。
  - 元金均等返済では、入力額を毎月の元金返済額として使用し、利息を加えた額がお支払い額になります。
  - ボーナス払いを使用する場合、入力額はボーナス払い分を除く通常返済部分に適用します。
  - 一括返済では金額指定を使用せず、返済期間で設定します。

## 編集時の動作

ローン条件を編集して保存すると、返済スケジュール全体を再計算します。

- 同じ返済回数に紐づく入金失敗情報は引き継ぎます。
- 返済回数を短くし、該当する回数がなくなった入金失敗情報は引き継がれません。

## 注意事項

本アプリは入力された条件から返済予定を試算・管理するためのアプリです。実際の金融機関の約定計算、営業日、遅延損害金、保証料、手数料、繰上返済には対応していません。
