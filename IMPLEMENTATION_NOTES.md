# 実装メモ

## 画面共通仕様

- 全画面の基準フォントは Yu Gothic UI 12pt
- ローン登録・編集画面の入力コントロールは Yu Gothic UI 14pt
- DataGridViewの行高は34px、ヘッダー高は40px

## 主なクラス

- `Program.cs`
  - exe同階層のConfig / Data / Logs初期化
  - 起動日時ログ初期化（exeと同じ階層のLogsフォルダ）
  - JSON設定読み込み
  - SQLite初期化
  - 未処理例外ログ

- `Infrastructure/LoanRepository.cs`
  - SQLiteテーブル作成
  - ローンと返済スケジュールの保存
  - 入金失敗情報管理

- `Services/RepaymentCalculator.cs`
  - 3種類の返済方式（元利均等・元金均等・一括）
  - 元利均等・元金均等に追加できるボーナス払い
  - 月割り / 日割り
  - 1円未満の四捨五入（固定）
  - 月末・土日調整

- `Forms/MainForm.cs`
  - ローン一覧
  - 返済スケジュール
  - 入金失敗登録
  - 表示後の実サイズに基づくSplitContainer初期化

- `Forms/LoanEditForm.cs`
  - ローン登録・編集
  - 返済期間指定 / 毎月の金額指定
  - ボーナス払いチェックボックスと設定欄
  - 返済額（元金） / 利息 / お支払い額を分けた複数行の概算表示

## 配布確認

配布時は `bin\Release\net48` またはVisual Studioの出力フォルダを基準に、次を確認してください。

- `LoanManagerApp.exe`
- `LoanManagerApp.exe.config`
- `System.Data.SQLite.dll`
- SQLiteのネイティブDLL
- その他、出力された依存DLL

開発PC上のプロジェクトフォルダからDLLを手作業で選別せず、Release出力フォルダ一式を別フォルダへコピーして起動試験してください。

## 端数処理

- 端数処理方法の選択機能はありません。
- `RoundingHelper.RoundToYen` で1円未満を四捨五入します。
- 50銭未満は切り捨て、50銭以上は1円へ切り上げます。
- JSON設定およびSQLiteのローンテーブルには端数処理項目を保存しません。


## 1.0.5 変更内容

- 初期値: 名称「無題」、借入額1,000,000円、年利15%
- 年利入力は最大2桁まで許可し、整数値は小数点なしで表示
- 返済期間指定と毎月返済額指定を選択可能
- 端数処理の画面表示を削除
- Config / Data / Logsをexeと同じ階層に保存
- 旧DB互換処理なし


## 1.0.6 変更内容

- ボーナス払いを返済方式からチェックボックスへ分離
- 元利均等返済と元金均等返済の両方でボーナス払いを利用可能
- 返済方式に括弧付き説明を追加
- `RepaymentSchedule.RepaymentAmount` を追加
- 返済額は通常返済分、お支払い額はボーナス払いを含む実支払合計として表示
- 旧DB互換処理なし。新しいDBスキーマのみを使用


## 1.0.7 変更内容

- 返済額を元金返済分、お支払い額を元金と利息の合計へ再定義

- 元金均等の金額指定は毎月の元金返済額として計算
- 返済スケジュールの重複していた元金列を画面から削除
- 概算欄を4行化
- 入力欄を14ptへ拡大し、フォーム・入力コントロールを調整


## 1.0.8 変更内容

- `LoanRepository.ReadLoan` で `PrincipalAmount` を読み込んでいなかった不具合を修正

- 登録・編集画面とローン詳細では従来どおり説明付き表示
